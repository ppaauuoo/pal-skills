# build-package

Three-step Windows packaging pipeline: PyInstaller → EXE, Inno Setup → installer, smoke test → verified.

## Prerequisites

- Python project with `pyinstaller` available (via `uv`, `pip`, or `conda`)
- Inno Setup 6.x — `iscc.exe` on `PATH` (or set `ISCC` env var to full path)
- `installer.iss` is auto-generated by this skill if absent (see Step 0 below)

## ⚠️ Pi execution environment (CRITICAL)

Pi's bash tool runs in **git-bash on Windows**. This affects every step:

| What breaks | Why | Fix |
|---|---|---|
| `uv run pyinstaller` | uv trampoline can't canonicalize script path in git-bash | `uv run python -m PyInstaller` |
| `cmd /c "..."` with output | Output never captured; often hangs | Write a temp `.ps1` file, run with `powershell -ExecutionPolicy Bypass -File <path>` |
| `start /B`, `timeout`, `tasklist` | cmd builtins — don't exist in git-bash | Use PowerShell (`Start-Process`, `Start-Sleep`, `Get-Process`, `Stop-Process`) via temp .ps1 |
| `where iscc` | cmd builtin | `which iscc 2>/dev/null` or check known path directly |
| `$variable` in inline powershell | bash interpolates `$` before powershell sees it | **Always** write to a .ps1 file first, never inline |

**Golden rule**: For anything beyond a simple command, write a temp `.ps1` script → run it → read output → delete it.

## Workflow

### Step 0 — Write `installer.iss` (if missing)

Before building, check whether `installer.iss` exists in the project root.

If it does **not** exist, inspect the project to fill in the template variables:

| Variable | Where to find it |
|----------|------------------|
| `AppName` | Project folder name, `pyproject.toml` `[project] name`, or ask the user |
| `AppVersion` | `pyproject.toml` `[project] version`, `__version__` in entry file, or `"1.0"` |
| `AppPublisher` | `pyproject.toml` `[project] authors`, or omit the line |
| `AppURL` | `pyproject.toml` `[project] urls`, or omit the line |
| `ExeName` | `<AppName>.exe` (matches `--name` passed to PyInstaller) |
| `IconFile` | Any `.ico` found under `assets/`, `resources/`, or project root; omit line if none |

**MANDATORY — Ask the user these questions before writing `installer.iss`:**

1. **Install directory**: What should the default install path be? (e.g. `{autopf}\<AppName>`, `C:\CVAD`, a custom path)
2. **Can the user change the install path?** If yes, add `DisableDirPage=no` to always show the directory picker — even on reinstalls.

Always include `DisableDirPage=no` unless the user explicitly says to lock the path.

Then **write** `installer.iss` using the canonical template:

```iss
; Auto-generated by build-package skill
[Setup]
AppName=<AppName>
AppVersion=<AppVersion>
AppPublisher=<AppPublisher>
AppPublisherURL=<AppURL>
AppSupportURL=<AppURL>
DefaultDirName=<DefaultDir>           ; ask user for this value
DisableDirPage=no                      ; always show dir picker so user can change path
DefaultGroupName=<AppName>
OutputDir=installer
OutputBaseFilename=<AppName>_Setup
Compression=lzma
SolidCompression=yes
WizardStyle=modern
; SetupIconFile=<IconFile>   ; uncomment if icon exists

[Languages]
Name: "english"; MessagesFile: "compiler:Default.isl"

[Tasks]
Name: "desktopicon"; Description: "{cm:CreateDesktopIcon}"; GroupDescription: "{cm:AdditionalIcons}"; Flags: unchecked

[Files]
Source: "dist\<AppName>\*"; DestDir: "{app}"; Flags: ignoreversion recursesubdirs createallsubdirs

[Icons]
Name: "{autoprograms}\<AppName>"; Filename: "{app}\<ExeName>"
Name: "{autodesktop}\<AppName>"; Filename: "{app}\<ExeName>"; Tasks: desktopicon

[Run]
Filename: "{app}\<ExeName>"; Description: "{cm:LaunchProgram,<AppName>}"; Flags: nowait postinstall skipifsilent
```

After writing, confirm the path: `installer.iss` created.

### Step 0.5 — Write `build_package.bat` (if missing)

Check whether `build_package.bat` exists in the project root. If not, write it now.
This bat is the single command a developer runs for all future releases — edit only
the three variables at the top when bumping a version.

Fill in the variables by inspecting the project (same sources as Step 0):

| Variable | Where to find it |
|---|---|
| `NAME` | `AppName` from `installer.iss` / PyInstaller `--name` |
| `VERSION` | `pyproject.toml` `[project] version` |
| `ISCC` | Run `which iscc 2>/dev/null`; if not on PATH check `$LOCALAPPDATA/Programs/Inno Setup 6/ISCC.exe` or `$HOME/AppData/Local/Programs/Inno Setup 6/ISCC.exe` |

**Detect onefile vs onedir** by checking whether the `.spec` has a `COLLECT` step or whether `installer.iss` `[Files]` sources `dist\<Name>.exe` (onefile) vs `dist\<Name>\*` (onedir). Choose the correct `del` vs `rmdir` clean command and the correct EXE path.

Write `build_package.bat` using this template, filled with the project values:

```bat
@echo off
setlocal
cd /d "%~dp0"

REM ── Edit these three lines when bumping a version ─────────────────────────
set NAME=<AppName>
set VERSION=<AppVersion>
set ISCC=<full path to ISCC.exe, or just: iscc>
REM ─────────────────────────────────────────────────────────────────────────

set INSTALLER=installer\%NAME%_Setup_v%VERSION%.exe
set TESTDIR=%TEMP%\%NAME%_test_install

echo ========================================
echo  %NAME% v%VERSION% — Build Pipeline
echo ========================================
echo.

REM ── 1/3  PyInstaller ──────────────────────────────────────────────────────
echo [1/3] Building EXE ...

REM onefile: del dist\<Name>.exe    |   onedir: rmdir /s /q dist\<Name>
<clean command>

REM If a .spec file exists, use it; otherwise use flags directly:
uv run python -m PyInstaller <Name>.spec --noconfirm
REM  — OR for a fresh onedir build without a spec:
REM uv run python -m PyInstaller --noconfirm --clean --name "%NAME%" --onedir --windowed --paths src src\main.py

if errorlevel 1 ( echo [ERROR] PyInstaller failed. & pause & exit /b 1 )
echo [OK] EXE built.
echo.

REM ── 2/3  Inno Setup ───────────────────────────────────────────────────────
echo [2/3] Packaging installer ...
"%ISCC%" installer.iss
if errorlevel 1 ( echo [ERROR] Inno Setup failed. & pause & exit /b 1 )
if not exist "%INSTALLER%" ( echo [ERROR] %INSTALLER% not found. & pause & exit /b 1 )
echo [OK] %INSTALLER%
echo.

REM ── 3/3  Smoke tests ──────────────────────────────────────────────────────
echo [3/3] Smoke-testing EXE (must stay alive 5 s) ...
start "" /B "dist\%NAME%\%NAME%.exe"
timeout /t 5 /nobreak >nul
tasklist /FI "IMAGENAME eq %NAME%.exe" 2>nul | find /I "%NAME%.exe" >nul
if errorlevel 1 ( echo [ERROR] EXE exited within 5 s -- startup crash. & pause & exit /b 1 )
taskkill /F /IM "%NAME%.exe" >nul 2>&1
echo [OK] EXE alive after 5 s.
echo.

echo [3/3] Smoke-testing installer (silent install) ...
if exist "%TESTDIR%" rmdir /s /q "%TESTDIR%"
"%INSTALLER%" /VERYSILENT /SUPPRESSMSGBOXES /NORESTART /DIR="%TESTDIR%"
if errorlevel 1 ( echo [ERROR] Installer returned non-zero. & pause & exit /b 1 )
if not exist "%TESTDIR%\%NAME%.exe" ( echo [ERROR] Installed EXE not found. & pause & exit /b 1 )
echo [OK] Installed EXE found.

REM Optional: run uninstaller to clean up registry after test
if exist "%TESTDIR%\unins000.exe" (
  "%TESTDIR%\unins000.exe" /VERYSILENT /SUPPRESSMSGBOXES /NORESTART
  timeout /t 3 /nobreak >nul
)
if exist "%TESTDIR%" rmdir /s /q "%TESTDIR%"

REM ── Summary ───────────────────────────────────────────────────────────────
echo.
echo ========================================
echo  BUILD COMPLETE
echo ========================================
for %%F in ("%INSTALLER%") do echo  Deliverable: %%~fF  (%%~zF bytes)
echo.
pause
endlocal
```

After writing, confirm the path: `build_package.bat` created.

> **Version bump checklist** — when releasing a new version, update these three files:
> 1. `pyproject.toml` — `version = "x.y.z"`
> 2. `installer.iss` — `AppVersion` and `OutputBaseFilename`
> 3. `build_package.bat` — `set VERSION=x.y.z`

### Step 1 — Build EXE with PyInstaller

**Pi execution** (git-bash):

```bash
# Clean previous build
rm -rf "dist/<AppName>"

# Build — ALWAYS use python -m PyInstaller, never bare pyinstaller
uv run python -m PyInstaller <AppName>.spec --noconfirm
# OR without a .spec:
uv run python -m PyInstaller --noconfirm --clean --name "<AppName>" --onedir --windowed --paths src "<entry_point.py>"
```

Common additions:

| Flag | Purpose |
|------|---------|
| `--add-data "config;config"` | Bundle a data folder (src;dest, Windows separator `;`) |
| `--collect-all <pkg>` | Pull in all files for a package (needed for GUI libs, cv2, etc.) |
| `--hidden-import <mod>` | Force-include modules PyInstaller misses |
| `--exclude-module <mod>` | Strip heavy unused packages (torch, onnxruntime, etc.) |
| `--contents-directory .` | Flatten resources into dist root instead of `_internal\` |
| `--onefile` | Single EXE alternative (slower startup, simpler distribution) |

#### uv projects

```bash
uv run python -m PyInstaller <flags> entry.py
```

#### pip / venv projects

```bash
.venv/Scripts/python -m PyInstaller <flags> entry.py
```

### Step 2 — Package installer with Inno Setup

**Pi execution** (git-bash):

```bash
# Find ISCC — try which first, fall back to known location
ISCC=$(which iscc 2>/dev/null || echo "$LOCALAPPDATA/Programs/Inno Setup 6/ISCC.exe")
"$ISCC" installer.iss
```

If `which` fails and the env var path has spaces, use the full path directly:

```bash
"C:/Users/<user>/AppData/Local/Programs/Inno Setup 6/ISCC.exe" installer.iss
```

Output: path defined in `installer.iss`, typically `installer\<AppName>_Setup.exe`.

### Step 3 — Test the Package

After both build steps succeed, run the smoke tests below **in order**. Stop and report on first failure.

**⚠️ Pi execution**: cmd builtins (`start`, `timeout`, `tasklist`) don't work in git-bash. **Always write a temp .ps1 script** and run it with `powershell -ExecutionPolicy Bypass -File <path>`. Never put PowerShell inline in bash (bash eats `$`).

#### 3a — EXE smoke test

Launch the EXE and verify it stays alive for at least 5 seconds (catches immediate crashes, missing DLLs, import errors).

**Pi execution** — write this to a temp file like `_smoke_exe.ps1`:

```powershell
$p = Start-Process '<full-path-to>\dist\<AppName>\<AppName>.exe' -PassThru
Start-Sleep 5
if (-not $p.HasExited) {
    Write-Output '[OK] EXE alive after 5s'
    Stop-Process $p -Force
} else {
    Write-Output '[FAIL] EXE exited within 5s'
    exit 1
}
```

Then run:

```bash
powershell -ExecutionPolicy Bypass -File _smoke_exe.ps1
rm _smoke_exe.ps1
```

For **console apps with `--version`/`--help`**, a simpler check works directly in bash:

```bash
"dist/<AppName>/<AppName>.exe" --version && echo "[OK]" || echo "[FAIL]"
```

#### 3b — Installer silent-install test

Install silently to a temp directory and verify the EXE lands where expected.

**Pi execution** — write to `_smoke_installer.ps1`:

```powershell
$testdir = "$env:TEMP\<AppName>_test_install"
if (Test-Path $testdir) { Remove-Item $testdir -Recurse -Force }

$installer = '<full-path-to>\installer\<AppName>_Setup.exe'
Start-Process $installer -ArgumentList '/VERYSILENT','/SUPPRESSMSGBOXES','/NORESTART',"/DIR=$testdir" -Wait

if (Test-Path "$testdir\<AppName>.exe") {
    Write-Output '[OK] Installed EXE found'
} else {
    Write-Output '[FAIL] Installed EXE not found'
    exit 1
}

# Cleanup — fire and forget uninstaller (don't -Wait, it's slow)
if (Test-Path "$testdir\unins000.exe") {
    Start-Process "$testdir\unins000.exe" -ArgumentList '/VERYSILENT','/SUPPRESSMSGBOXES','/NORESTART'
    Start-Sleep 3
}
if (Test-Path $testdir) { Remove-Item $testdir -Recurse -Force -ErrorAction SilentlyContinue }
Write-Output '[OK] Cleanup done'
```

Then run:

```bash
powershell -ExecutionPolicy Bypass -File _smoke_installer.ps1
rm _smoke_installer.ps1
```

#### 3c — Installed EXE re-launch check (optional)

Same pattern as 3a but pointing at the installed copy. Only run if 3b left `$testdir` intact (skip cleanup in 3b, do it after 3c).

## Outputs

| Path | Description |
|------|-------------|
| `dist\<AppName>\<AppName>.exe` | Standalone EXE (one-dir) |
| `dist\<AppName>.exe` | Standalone EXE (one-file) |
| `installer\<AppName>_Setup.exe` | Windows installer (Inno Setup) |

## Always rebuild clean

Delete `dist\<AppName>\` before each build to avoid stale files leaking into the package:

```bash
rm -rf "dist/<AppName>"
```

## Common failures

| Error | Fix |
|-------|-----|
| `ModuleNotFoundError` at runtime | Add `--hidden-import <module>` |
| DLL / `.pyd` missing at runtime | Add `--collect-all <package>` |
| `iscc` not found | Install Inno Setup 6.x; verify with `which iscc` or check `$LOCALAPPDATA/Programs/Inno Setup 6/ISCC.exe` |
| `uv trampoline failed to canonicalize script path` | **Never** use `uv run pyinstaller`. Use `uv run python -m PyInstaller` |
| Build succeeds but EXE crashes silently | Drop `--windowed` temporarily to see console errors |
| `dist` too large | Add `--exclude-module` for unused heavy deps (torch, scipy, etc.) |
| Resources not found at runtime | Check `--add-data` paths; use `sys._MEIPASS` in code for one-file mode |
| EXE exits in <5 s during smoke test | Run without `--windowed` to capture traceback; fix missing imports/data |
| Installer test: `/VERYSILENT` hangs | UAC prompt needs elevation — run test from an elevated shell |
| `cmd /c` output not captured in pi | pi runs git-bash; cmd /c swallows stdout. Write a .ps1 and run that instead |
| `$variable` eaten by bash in inline PS | Never inline PowerShell in bash. Write to .ps1 file, then `powershell -ExecutionPolicy Bypass -File` |
| Smoke test powershell timeout on uninstaller cleanup | Uninstaller can be slow; don't `-Wait` on uninstall, use `-ErrorAction SilentlyContinue` |
| Installed EXE not found at expected path | Check `DefaultDirName` in `installer.iss`; adjust `TESTDIR` to match |
